# Generic Kubernetes And Jenkins Platform Template

English | [한국어](README.ko.md)

This repository is a reusable platform template for teams that want generic Kubernetes manifests, optional Jenkins automation, and public-image application examples without carrying private company services or fixed company port conventions.

## What Changed

- Private application images were removed from the default path.
- Application examples now use public images:
  - `nginx-web`
  - `httpbin`
  - `whoami`
  - `adminer`
- Old `31500` range ports were replaced with common service ports such as `80`, `8080`, `3306`, and `5432`.
- Environment values are now easy to edit through `config/platform-values*.env` and `config/service-runtime.env.example`.

## Repository Layout

- `config/`: editable values files, profiles, environment presets, and service catalogs
- `jenkins/`: generic repository-level Jenkins jobs for validation, delivery, promotion, and seed automation
- `k8s/`: Kubernetes manifests grouped by rollout phase and concern
- `scripts/`: rendering, validation, planning, and bundle helper scripts
- `services/`: local Docker Compose examples based on public images

Each top-level folder and each application/component folder now has its own `README.md`.

## Generic Application Examples

- `nginx-web`: static site example using `nginx:1.28-alpine`
- `httpbin`: HTTP test endpoint using `mccutchen/go-httpbin:v2.15.0`
- `whoami`: request inspector using `traefik/whoami:v1.10.4`
- `adminer`: database UI using `adminer:5.3.0-standalone`

## Typical Workflow

1. Compare profiles.

```powershell
.\scripts\show-profile-catalog.ps1 -Format markdown
```

2. Generate or copy an environment values file.

```powershell
.\scripts\new-platform-environment.ps1 -EnvironmentPreset dev -EnvironmentName dev
```

3. Preview the selected bundle.

```powershell
.\scripts\show-platform-plan.ps1 `
  -Profile web-platform `
  -Applications nginx-web,httpbin,whoami `
  -DataServices redis `
  -Format markdown
```

4. Validate the repository and rendered assets.

```powershell
.\scripts\invoke-repository-validation.ps1 -EnvironmentPreset dev
```

5. Render and archive a bundle.

```powershell
.\scripts\invoke-bundle-delivery.ps1 -EnvironmentPreset dev
```

## Notes

- Public images are used by default, so a private registry is not required unless you customize the template.
- Example hostnames use `example.com` and should be replaced.
- Generated values files are intended to be edited directly by whoever adopts the template.
- Jenkins service-image jobs are optional and not generated by default for these public-image examples.
