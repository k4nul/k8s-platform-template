# Kubernetes 매니페스트

[English](README.md) | 한국어

이 디렉터리에는 배포 단계와 성격에 따라 나눈 재사용 가능한 Kubernetes 매니페스트가 들어 있습니다.

## 번호 규칙

- `100_*`: 기반 인프라
- `200_*`: 스토리지
- `300_*`: 공용 서비스와 클러스터 플랫폼 컴포넌트
- `400_*`, `401_*`: 애플리케이션 예제

이 번호는 단순한 장식이 아니라, 디렉터리의 의도와 대략적인 배포 흐름을 보여주는 역할도 합니다.

## 배포 단계 모델

저장소 차원의 계획 문서에서는 좀 더 큰 단계로 나눠서 봅니다.

- 기반 인프라
- 클러스터 애드온
- 공용 서비스
- 애플리케이션
- 컨트롤러 준비 후 나중에 적용하는 deferred 리소스

예를 들어 Gateway API 관련 리소스는 `300_*` 범위에 있어도 실제 적용 시점은 더 나중으로 잡힐 수 있습니다.

## 이 매니페스트를 사용하는 권장 방식

특별한 이유가 없다면, 저장소의 원본 디렉터리를 바로 `kubectl apply` 하는 것보다 아래 흐름이 더 안전합니다.

1. `config/` 아래 값 수정
2. `show-platform-plan.ps1` 로 포함 대상을 확인
3. 번들 렌더링
4. 렌더링된 `out/` 결과 검토
5. 렌더링된 번들 기준으로 적용

예시:

```powershell
.\scripts\show-platform-plan.ps1 -Format markdown
```

## 이 디렉터리에서 찾을 수 있는 것

- 네임스페이스와 스토리지 같은 공용 기반 리소스
- MySQL, PostgreSQL, Redis, Memcached, NGINX 같은 플랫폼 서비스
- ExternalDNS, Harbor, Longhorn, Dashboard, VPA 같은 선택형 애드온 values 스캐폴드
- 공개 이미지 기반 샘플 애플리케이션

각 컴포넌트 디렉터리에는 개별 `README.md` 가 있어서, 그 디렉터리에서 바로 필요한 설명을 볼 수 있습니다.
